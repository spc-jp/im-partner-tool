— 01 / BACKGROUND

時代背景と現状認識

Webサイトは「構築するもの」から「経営とともに育つもの」へと役割が変わりました。
御社の30,000アカウント基盤の「先にある課題」は、二つの現場で同時に進行しています。

代理店 / 売る側

「Webの専門知識が、提案の壁になっている。」

顧客の生の悩み（採用、集客、ブランド）を、システム要件に翻訳できる人材が枯渇。
結果、「何を売っていいかわからない」状態のパートナーが眠ったままになっています。

中小企業 / 使う側

「導入したのに、使いこなせず放置。」

専門用語ばかりの管理画面、無機質なアクセス数のグラフ。
「次に何をすればいいのか」がわからず、更新が止まり、やがてチャーン（解約）へ。

— 02 / STRATEGY

戦略の3本柱

本構想は、機能追加ではなく「事業構造そのもの」をアップデートする提案です。
売る・使う・続けるの3つの局面を、ひとつの思想で貫きます。

01
SELL

売る仕組み

顧客の悩みを入力するだけで、AIが最適パッケージと提案トークを自動生成。
「誰でも売れる」を仕組みにします。

02
SELECT

創るから選ぶへ

EC・予約・多言語などのオプションを、アプリストアのように「選んでオン」。
受託開発依存から脱却します。

03
STAY

心理の理解

「お疲れさまです、今週は採用ページが伸びています」。
無機質なグラフではなく、温かい伴走で離脱を防ぎます。

— 04 / CORE FEATURES

コア機能の設計

3つの柱は、3つの実装機能としてアプリに落とし込まれます。

FEATURE 01

AIヒアリング＆自動パッケージ提案

代理店がタブレットを片手に顧客と話す。「採用に困っている」「近所の人に来てほしい」――
そのままの言葉をAIが受け取り、ベースとなる New CCM に最適なモジュール（求人 / エリアSEO / 予約 等）を組み合わせ、
「選ぶだけのパッケージ構成」と「なぜ必要かの提案書」を即座に出力します。

— 05 / INSIGHT

マーケットインサイトの可視化

「なぜ売れたか・なぜ売れなかったか」を、経営の意思決定に直結するデータへ。

FOR EXECUTIVES

失注・解約理由を、AIタグ付けで経営ダッシュボードへ。

代理店が入力した失注理由、顧客の解約時アンケートをAIが分類。
「価格」「機能不足」「サポート不足」「業種特有の課題」などのタグで集計し、
御社の経営陣に「次のパッケージ開発の意思決定」を、リアルタイムで届けます。

── 営業現場のリアルな声が、商品戦略へ直接フィードバックされる構造。
御社の20年の経験を、これからの20年のアルゴリズムに変換します。

— 07 / POSITIONING

これは「次世代CMS」ではない。

機能の足し算ではなく、「Webサイトと中小企業経営の関係」を再設計する事業構想です。
御社の強み――30,000件の実績、自社開発CMS、全国の販売パートナー網――の上にしか成立しません。

A
ASSET

資産の上に立つ。

20年・3万件のデータベース「SmartDB」が、AIパッケージ提案エンジンの精度を生みます。
競合は、この学習データを持ちません。

B
CHANNEL

チャネルが武器になる。

既存パートナー網が「売れない」課題を抱えた瞬間、本プラットフォームは即戦力に。
網そのものの収益性が変わります。

C
PHILOSOPHY

理念が、差別化になる。

「中小企業の元気を支える」という御社のミッションを、機能ではなく顧客体験として体現します。

— 08 / ROADMAP

導入ロードマップ（案）

最小構成での実証 → 段階的な機能拡張という、御社の文化に合った導入設計を想定しています。

PHASE 01

構想合意 ／ プロトタイプ設計

戦略の3本柱に対する経営陣の合意形成と、最小機能（AIヒアリング機能）のプロトタイプ着手。代理店数社へヒアリング。

PHASE 02

パイロット ／ 代理店10社で実証

AIパッケージ提案機能を限定リリース。実営業現場で成約率・提案時間の変化を計測。改善ループを回します。

PHASE 03

オプションストア／伴走UX 実装

機能オプション3〜5種を選んでオンにする仕組みを実装。並行して伴走型ダッシュボードのUXを既存顧客に展開。

PHASE 04

全パートナー展開 ／ インサイト機能稼働

全販売店へ本プラットフォームを展開。マーケットインサイト機能を経営ダッシュボードへ接続し、商品戦略のPDCAを回します。
